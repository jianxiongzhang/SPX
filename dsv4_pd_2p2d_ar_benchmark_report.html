AR OFF vs AR ON DDP ON — 参数如何传入

核心结论：整套 benchmark 只用一个 shell 变量 NCCL_AR_MODE 在两种模式间切换； 值为 ar_off 或 ar_on_ddp_on。脚本将其翻译为 4 个 NCCL IB 环境变量， 在每个 prefill/decode 进程启动前注入 docker 内的 sglang.launch_server 环境。 两种模式之间必须完整重启集群（stop → start），否则旧进程不会更换 NCCL 参数。

2. 翻译：NCCL_AR_MODE → 4 个 NCCL 变量

逻辑在 pd/env_pd_ep16_2p2d.sh 的 nccl_ar_overlay() 函数。 sglang_ncclep_block(mode) 会把该函数的 export 语句嵌入每个 job 的启动 shell。

ar_off（默认）0000AR_OFF

命名含义：AR ON = InfiniBand Adaptive Routing 开启； DDP ON = 同时开启 OOO RQ、Receiver-side Matching、Prepost Receive WR（与 ar_bench 3-case 中 ar_on_ddp_on 配置一致）。 ar_label() 仅用于日志/结果目录命名（AR_OFF / AR_ON_DDP_ON），不参与 NCCL 传参。

3. 注入点：参数进入哪些进程？

  • 受影响：4 个 sglang.launch_server 进程（2× prefill + 2× decode），每个节点 tp=8 ep=8 的 NCCLEP MoE 节点内 NCCL 通信
  • 不受影响：
    • KV 跨节点传输：走 NIXL/UCX（nixl_rdma_block()），与上述 4 个 NCCL AR 变量无关
    • Router（sglang_router.launch_router）：不加载 NCCL AR overlay，仅做 HTTP PD 路由
    • bench_serving 客户端：只连 router URL，不传 NCCL 参数

4. 两种模式如何隔离？

  1. bench_dsv4_ncclep_pd_2p2d_ar.sh 跑完 AR OFF 全部 case 后调用 run_dsv4_ncclep_pd_ep16_2p2d.sh stop（kill 全部 sglang 进程）
  2. 再 export NCCL_AR_MODE=ar_on_ddp_on 并 start，重新拉起 4 job + router
  3. 结果分别写入 .../AR_OFF/ 与 .../AR_ON_DDP_ON/ 目录；日志文件名含 AR_OFF / AR_ON_DDP_ON 后缀

用例设计（17 cases：13 baseline + 4 AR showcase）

用例集前 13 个与 dsv4_pd_1p1d_benchmark_report.html 对齐；组 F 为 2P2D AR 专项补充， 用于放大 NCCLEP MoE 节点内 NCCL 通信差异（详见「AR OFF vs AR ON DDP ON — 参数如何传入」）。

组 A: Decode 专项 — 短序列、高并发，主要测 decode 侧 MoE/NCCLEP 与 PD 解码吞吐

  • pd_decode_lat_c1 — ISL=128, OSL=128, conc=1: 短 prompt、单并发，隔离 decode 单请求 TTFT/TPOT 基线
  • pd_decode_thr_c8 — ISL=128, OSL=128, conc=8: 固定短序列，逐步拉高并发，测 decode 池饱和吞吐
  • pd_decode_thr_c16 — ISL=128, OSL=128, conc=16: 继续扩展并发，观察 decode 排队与 TPOT 劣化
  • pd_decode_thr_c32 — ISL=128, OSL=128, conc=32: 接近 decode 侧 max_running_requests，测峰值 PD 吞吐

组 B: 混合负载 — 典型 chat 比例 (ISL≈2K, OSL≈512)，测完整 PD 路径

  • pd_mixed_c1 — ISL=2048, OSL=512, conc=1: 中等 ISL + 中等 OSL，单并发端到端 PD 路径
  • pd_mixed_c4 — ISL=2048, OSL=512, conc=4: 混合负载并发，覆盖 prefill+decode+KV 协同
  • pd_mixed_c8 — ISL=2048, OSL=512, conc=8: 更高并发混合压测，观察 router 调度与 KV 传输

组 C: Prefill 专项 — 长输入，测 prefill 节点算力与 TTFT

  • pd_prefill_c1 — ISL=4096, OSL=128, conc=1: 长输入短输出，prefill 计算占主导
  • pd_prefill_c4 — ISL=4096, OSL=128, conc=4: prefill 并发，测 P 节点 batch 与 TTFT 扩展
  • pd_long_prefill_c1 — ISL=8064, OSL=128, conc=1: 接近 context 上限的 prefill，测长前缀 KV 生成

组 D: 长上下文 — ISL=8192 + OSL=1024，需 ctx≥10240、P mem=0.55（补跑专用配置）

  • pd_long_ctx_c1 — ISL=8192, OSL=1024, conc=1: ISL+OSL 接近上限，单并发长上下文稳定性
  • pd_long_ctx_c4 — ISL=8192, OSL=1024, conc=4: 长上下文并发，prefill OOM/KV 压力关键 case

组 E: KV 传输压力 — 多并发持续 KV RDMA，测 NIXL/IB 与 decode 侧接收

  • pd_kv_stress_c8 — ISL=4096, OSL=256, conc=8: 持续多请求 KV over IB（NIXL），测 RDMA 链路稳定性

组 F: AR 专项补充 — 高并发 decode/mixed + 长输出 decode，放大 NCCLEP MoE 节点内 NCCL 差异（推荐用于 AR showcase）

  • pd_decode_thr_c48 — ISL=128, OSL=128, conc=48: c32 之上继续加压，逼近每 decode 节点 max_running_requests=32
  • pd_mixed_c16 — ISL=2048, OSL=512, conc=16: 2P2D 四节点流水线中高并发 mixed，P+D MoE NCCL 同时繁忙
  • pd_mixed_c32 — ISL=2048, OSL=512, conc=32: 最高混合并发，最有希望拉开 AR OFF vs ON 的 Out/TPOT
  • pd_decode_long_c16 — ISL=128, OSL=512, conc=16: 短 prompt 长 OSL，增加每请求 decode MoE 步数与 NCCL 流量

长上下文补跑说明：组 D 中 case 在默认配置（ctx=8192, mem=0.70）下 prefill OOM； 补跑时使用 ctx=10240, mem-fraction-static=0.55, P_max_running_requests=4。

sync_deep_gemm_cache() {
  local src="$1" dst="$2"
  local marker
  marker="${LOGDIR}/deep_gemm_cache_synced_${dst}.done"
  if ssh_n "$dst" "test -f ${marker}"; then
    echo ">>> DeepGEMM cache already synced to ${dst} (skip)"
    return 0
  fi
  echo ">>> sync DeepGEMM cache ${src} -> ${dst} (container tar pipe, ~几秒)"
  ssh_n "$dst" "docker exec ${CONTAINER} bash -lc 'mkdir -p /root/.cache'"
  sshpass -p "$(pass_for "$src")" ssh -o StrictHostKeyChecking=no "root@${src}" \
    "docker exec ${CONTAINER} tar czf - -C /root/.cache deep_gemm 2>/dev/null" \
  | sshpass -p "$(pass_for "$dst")" ssh -o StrictHostKeyChecking=no "root@${dst}" \
    "docker exec -i ${CONTAINER} tar xzf - -C /root/.cache"
  ssh_n "$dst" "docker exec ${CONTAINER} bash -lc 'du -sh /root/.cache/deep_gemm 2>/dev/null; touch ${marker}'"
}

wait_health() {
  local host="$1" port="$2" name="$3" logfile="$4"
  local start
  start=$(date +%s)
  while [ $(($(date +%s) - start)) -lt 900 ]; do
    if ssh_n "$host" "curl -sf http://127.0.0.1:${port}/health >/dev/null 2>&1"; then
      echo "${name} ready on ${host}:${port}"
      return 0
    fi
    if ssh_n "$host" "test -f ${logfile} && tail -8 ${logfile} 2>/dev/null | grep -qE 'Scheduler hit an exception|OutOfMemory|NCCLEpError|EPException|Fatal Python error|Device API|shape mismatch'"; then
      echo "${name} crashed on ${host}:"
      ssh_n "$host" "tail -40 ${logfile}"
      return 1
    fi
    sleep 15
  done
  echo "${name} startup timeout on ${host}"
  ssh_n "$host" "tail -30 ${logfile} 2>/dev/null" || true
  return 1
}

def load_jsonl_metrics(path: Path) -> dict | None:
    if not path.exists():
        return None
    best = None
    for line in path.read_text().splitlines():
        line = line.strip()
        if not line:
            continue
        try:
            obj = json.loads(line)
        except json.JSONDecodeError:
            continue
        if "output_throughput" not in obj and "mean_ttft_ms" not in obj:
            continue
        if best is None or obj.get("completed", 0) > best.get("completed", 0):
            best = obj
    if not best or best.get("completed", 0) <= 0:
        return None
    tp = best.get("output_throughput", 0) or 0
    return {
        "ttft": best.get("mean_ttft_ms"),
        "tpot": best.get("mean_tpot_ms"),
        "out_tp": tp,
        "in_tp": best.get("input_throughput"),
        "per_gpu": tp / 32 if tp else None,
        "dur": best.get("duration"),
        "status": "ok",
    }

def delta_pct(base: float | None, val: float | None, lower_is_better: bool = False) -> str:
    if base is None or val is None or base == 0:
        return "—"
    pct = (val - base) / base * 100
    if lower_is_better:
        pct = -pct
    sign = "+" if pct > 0 else ""
    return f"{sign}{pct:.1f}%"


def cls_delta(pct_str: str) -> str:
    if pct_str == "—":
        return "neu"
    try:
        v = float(pct_str.rstrip("%").replace("+", ""))
    except ValueError:
        return "neu"
    if v > 2:
        return "win"
    if v < -2:
        return "neg"
    return "neu"

    on = load_mode_metrics(on_dir)

        if n.get("status") == "ok":
            ok_on += 1
        d_ttft = delta_pct(o.get("ttft"), n.get("ttft"), lower_is_better=True)
